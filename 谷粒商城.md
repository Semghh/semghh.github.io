# 1.分布式系统概述

![image-20220513194748677](C:\Users\SemgHH\AppData\Roaming\Typora\typora-user-images\image-20220513194748677.png)



注册中心

```
在分布式系统中，每个微服务上线都应该把自己注册到 注册中心。
```

配置中心

```
某一种微服务可能部署多个集群，如果要修改配置，同时需要修改多台集群的配置，非常麻烦。所以需要配置中心，集中管理配置
```

网关

```
前端所有的请求，都需要通过网关。进行统一的鉴权，过滤，路由.由网关再抵达各个服务。做一些预先的工作。
```

### 1.4.2 概念



#### 1.4.2.1 命名空间

namespace是 用于做配置隔离的。





```
默认保留的空间为public ， 默认新增所有配置都在 public中。
```



可以为不同的开发环境配置不同的命名空间

```
例如:
测试阶段 test
开发环境 dev
生产环境 prop
```



如何告诉机器使用哪个命名空间呢？

```
在bootstrap中配置
spring.cloud.nacos.config.namespace=d56c93ef-b2ce-453d-b1a2-b708e8342baa
```

![image-20220514104826849](C:\Users\SemgHH\Desktop\笔记\谷粒商城.assets\image-20220514104826849.png)
