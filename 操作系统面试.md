# 1. 操作系统基础







## 1.1 基础概念



什么是操作系统？

```
Operating System ，OS下组织管理计算机硬件，上服务用户请求的 系统软件。
```









操作系统内核做了什么?

```
硬件设备的调度
内存的管理
进程的调度
```







内核态/系统态/目态

```
真正不受限制的调度计算机硬件资源的状态。
```

用户态

```
用户态运行的进程可以直接读取用户程序的数据。 用户态需要请求使用额外的系统资源，需要向OS请求，并得到OS的响应，由用户态转为内核态执行对应请求。
```



什么是系统调用？

```
首先说明什么是用户态，和 内核态。

为什么要有用户态和内核态区分？  为了系统的安全，用户态不能随意操作系统敏感关键的权限。不慎操作将造成系统崩溃


与系统态级别的资源有关的操作（如文件管理、进程控制、内存管理等)，都必须通过系统调用方式向操作系统提出服务请求，并由操作系统代为完成。
```

![image-20220323174505110](操作系统面试.assets/image-20220323174505110.png)





cpu时间片轮换制度

```
cpu通过时分复用，在较短的持续时间内，将cpu时间片轮流的分给各个线程/进程使用。

通过多个线程共享时间片,达到并发的效果。
```











## 1.2  进程与线程



进程

```
系统分配资源、和保护的基本单位。 一个进城可以拥有多个线程。
```

线程

```
系统调度的基本单位。比进程小，属于同一个进程的线程共用进程的一部分资源。

切换、创建、调度的开销比进程小。

在编程中使用线程工作，符合单一职责原则，符合解耦思想。
```





进程的几种状态

```
创建状态  :  进程新创建的状态	
等待状态/阻塞状态  : 进程在等待某种状态，例如IO
就绪状态  :   进程获得全部所需的资源。等待cpu时间片的分配
运行状态  :   这个进程已经获得了cpu时间片
终止状态  :   进程结束、凋亡
```



什么是临界区？

```
多个线程共享的资源称为临界区
```



### 进程同步方式

```
1. 信号量。 是一个同步对象，具有2种操作， P 和 V 原语。对应Java种Semaphore类。 P操作会减少信号量，V会增加信号量。想要进入临界区的进程需要调用P原语，当信号量为负时进程会阻塞等待，当信号量不为负可以进入临界区。


2.互斥量 ，可以理解为值只有0和1的信号量。当进程想要进入同步代码块时，需要获得互斥量。当持有互斥量时，阻止其他进程进入临界区


3.管程(Monitor) 是一种程序结构,内部有一定的数据结构。外部想要进入临界区的进程无需知道管程内部结构。管程实现了在统一时刻只有1个进程访问临界区。管程支持条件变量，对应着java的Condition类，条件变量就像一个断言，当满足断言时，等待在这个条件变量的进程可以唤醒，表示占有当前管程。


```







### 进程之间通信的方式

```
1.管道pipeline:  也称匿名管道。半双工通信方式,数据只能单向流动。只能用于亲缘关系(父子进程)的进程之间使用。被创建的进程称为子进程，反过来创建子进程的称为父进程。

2.	有名管道 ： 有名管道以磁盘文件的方式存在，可以实现本机任意两个进程通信。
	
	
	
3.信号量 ：  是一个计数器，用于多线程之间通信。



4.信号 : 当事件发生时，对进程的通知机制。也成为软件中断，在软件层次上的中断机制。





5.套接字 socket : 此方法主要用于在客户端和服务器之间通过网络进行通信。套接字是支持 TCP/IP 的网络通信的基本操作单元，可以看做是不同主机之间的进程进行双向通信的端点



6.共享内存 : 多个进程共享同一块内存，对内存操作。

            操作系统如何知道一块共享内存被多少个进程关联？
            答：	共享内存维护了一个结构体struct shmid_ds，
                这个结构体中有一个成员 shm_nattch，
                shm_nattach 记录了关联的进程个数


```





















线程之间同步的方式

```
1. 互斥量 :  拥有互斥量的线程才能访问临界区。 也就是锁 例如java synchronized Reentrantlock
2. 信号量 :  规定进入临界区线程的数量。 对应 java Semaphore
3. 事件驱动 :  例如java  Object.wait() Object.notify()
```













## 1.3  进程调度算法



### 1.3.1 FIFO 先进先出

```
根据就绪队列里进程 进入队列的顺序，依次被cpu调度
```

优点：

```
简单，不会造成饥饿现象。
```

缺点：

```
没有优先级。重要的程序不能被优先执行。
```







### 1.3.2 SJF最短作业优先

```
从就绪队列中选出一个预估最短的作业优先被调度。
```

优点:

```
任务量吞吐率高
```

缺点：

```
多数作业无法合理预估时间、长时间作业将被搁置。造成饥饿现象
```

## 1.4 死锁



### 1.4.1 什么是死锁？

多个进程之间相互争抢资源。最终谁都无法获得足够资源执行下去。都在等待其他进程释放资源，而无限期等待下去。





### 1.4.2 形成死锁的条件



```
互斥 		 :   资源被进程独享
占有并等待 : 占有资源，并等待所有资源运行。
非抢占    :  不会抢占其他线程的资源
循环依赖  : 等待的资源形成循环依赖等待。
```



### 1.4.3 解决死锁方法



```
预防 :  限制并发进程
避免 :  在分配资源时，提前预测
检测 :  专门检测死锁的发生，定位死锁线程，和等待的资源
解决 :  解除死锁
```



## 1.5  内存管理



操作系统的 【内存管理】 主要做什么？

```
1. 内存的分配和回收
2. 逻辑地址转化为真实的物理地址
```



程序的局部性原理是什么？

```
空间局部性： 某个被访问的存储单元附近的存储单元在未来有很大概率被访问。
时间局部性： 某个被访问的存储单元在未来有很大概率被再次访问。
```











### 1.5.1 内存管理策略



#### 1.5.1.1 实存管理

```
实存管理要求，程序必须都全部加载到内存中。

这显然是未被程序局部性原理的，浪费了大量的内存空间。


实存管理又分为: 连续分配、非连续分配。
```



``` 
连续分配管理    :  固定分区:
				 可变分区:按照作业大小划分。划分的时间、大小、位置都是可变的，动态的。
				
				
非连续分配管理  :  页式管理 、
				段式管理 、
				段页式管理
```

```
连续和非连续的最大区别就是，非连续允许程序占有一个或多个不连续的空间。
```

##### 1.5.1.1.2 页式分配管理

![image-20220323183848204](操作系统面试.assets/image-20220323183848204.png)

```
逻辑上的内存分区： 页面
物理地址上的内存分区: 页框  
在大小上：页面=页框

一个逻辑地址:  由两部分组成，   页号+页内偏移量


页表 ：  用于逻辑地址向物理地址转换
```





###### 快表

```
如果将页表存储在内存，则访问速度过慢。
若果将页表全部存储在一组寄存器中，硬件成本过于高昂。

所以在硬件中设置了 相连寄存器，用于缓存最近使用过的页表。 (就相当于硬件级别的缓存)
```



当逻辑地址想要映射为物理地址的过程：



根据页号找快表，如果找到直接映射为物理地址。

如果没有，访问内存中的页表，

并把这词页号对应的页框号刷新到快表中。

快表即将满了，根据算法淘汰部分记录。





















##### 1.5.1.1.3 段式分配管理



```
页式管理仅仅按照页面大小把内存划分。不含有任何实际意义，段式管理的出现是为了面向程序员。每一个段的划分。每一个段的分配，都具有实际意义。


页面的分段，与源程序没有任何关联。很容易出现一个正在运行的程序端突然用光页了。此时不得不停下来，进行极其耗时的磁盘IO操作,把下一页加载到内存中。这样的设计不符合 “程序局部性原理”。

所以,让内存分配具备“现实意义”的段式管理出现了。
```



![image-20220323183852583](操作系统面试.assets/image-20220323183852583.png)



````
和页表结构类似

段表 ：  为了映射虚拟地址到真实地址。 映射了段号到真实物理地址

真是物理地址  =  段号+段内偏移量
````

##### 1.5.1.1.4 段页式管理



 段页式管理机制

```
段页式管理机制结合了段式管理和页式管理的优点。简单来说段页式管理机制就是把主存先分成若干段，每个段又分成若干页，也就是说 段页式管理机制 中段与段之间以及段的内部的都是离散的。
```





#### 1.5.1.2  虚存管理

```
虚存允许先加载程序的一部分到内存中并运行。当实际运行时发生缺页，则会发生IO。
```



```
优点: 提高了内存利用率
缺点: 当发生缺页现象时，会发生磁盘IO，降低程序运行效率
```



缺页现象

```
程序请求的数据不在内存中。操作系统不得不进行磁盘IO把数据写入到内存供程序继续使用
```



缺页中断率

 ```
 中断率 = 缺页次数/总访问次数
 ```

#### 1.5.1.3  cpu寻址，为什么要使用虚拟地址



现代处理器使用的是一种称为 **虚拟寻址(Virtual Addressing)** 的寻址方式。

所谓虚拟寻址，就是cpu需要把虚拟地址转化为物理地址再去访问。

cpu中的一个硬件完成转换功能。



为什么要使用？

```
直接操作地址值有可能带来数据的不安全。
例如其他进程由于程序错误，直接操作了其他进程的内存。这将是灾难级的后果。
```

```
同时，虚拟地址允许程序可以使用连续的虚拟地址来映射实际不相连的物理地址。
不用进程使用虚拟地址，彼此相互隔离。
```





## 1.6  虚拟内存

通过页面替换技术，扩展内存的一种方式。可以让程序拥有远超过实际物理内存的可用内存。

```
多任务系统,必然会带来的问题就是内存不够。而内存的单价相较于磁盘是高昂的，同时处理器也存在支持的内存上限。根据程序局部性原理，程序对内存的90%访问只局限于10%的空间。有即调用缓存、没有则加载的的虚拟内存技术就应运而生了。
```
