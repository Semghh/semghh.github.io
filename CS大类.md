

# 1. SAAS



## 34.1 什么是SAAS

百度百科 ：SaaS，是Software-as-a-Service的缩写名称，意思为软件即服务，即通过网络提供软件服务。 



说人话就是，不再为某一个大客户做专门的定制软件开发服务。而是将多个客户之间共有的需求，做成一个共性的【服务】卖给多个客户。

例如`广告搜索服务` ，用户的购买形式也会有多种变化。不再是买断整个软件的代码，而是购买服务的使用时长/使用次数/使用流量等。





















# 2. Qos

`Qos(Quality of Service)` 服务质量。指一个网络能够利用各种基础技术，为指定的网络通信提供更好的服务能力，是网络的一种安全机制， 

是用来解决[网络延迟](https://baike.baidu.com/item/网络延迟/10680325?fromModule=lemma_inlink)和【阻塞】等问题的一种技术。





【QoS的保证】对于容量有限的网络来说是十分重要的，特别是对于【流多媒体】应用，例如[VoIP](https://baike.baidu.com/item/VoIP/110300?fromModule=lemma_inlink)和IPTV等，因为这些应用常常需要固定的[传输率](https://baike.baidu.com/item/传输率/10811244?fromModule=lemma_inlink)，对延时也比较敏感。







`Qos` 是关心网络质量的一类总称。 不同的网络质量优化的手段，可能位于不同的OSI层，工作原理也不相同。









## 35.1 问题提出

当网络发生拥塞的时候，所有的[数据流](https://baike.baidu.com/item/数据流?fromModule=lemma_inlink)都有可能被丢弃； （交换机拥塞时可丢弃数据包）

为满足用户对不同应用不同服务质量的要求，就需要网络能根据用户的要求分配和调度资源，对不同的数据流提供不同的服务质量：

```
对实时性强且重要的数据报文优先处理；

对于实时性不强的普通数据报文，提供较低的处理优先级，网络拥塞时甚至丢弃。
```













## 35.2  解决

QoS应运而生。支持QoS功能的设备，能够提供传输品质服务；针对某种类别的数据流，可以为它赋予某个级别的传输优先级，来标识它的相对重要性，并使用设备所提供的各种优先级转发策略、拥塞避免等机制为这些数据流提供特殊的传输服务。





配置了QoS的[网络环境](https://baike.baidu.com/item/网络环境?fromModule=lemma_inlink)，增加了网络性能的可预知性，并能够有效地分配网络[带宽](https://baike.baidu.com/item/带宽?fromModule=lemma_inlink)，更加合理地利用网络资源。









# 3. 内网穿透



`ipv4`地址不够用的背景下， 会出现多台计算机使用同一个 ip地址的情况。那么如何在一个`ip地址`下隔离这些计算机呢？ `NAT`技术。

`NAT` 对内是一个私有的局域网，对外仍是同一个`ip`的整体，这导致了外网无法主动和 `NAT` 内的主机沟通，只能由`NAT`内的主机主动建立连接。



那么`内网穿透` 实际上就是 `NAT穿透` 。

# 4. 快速求一个大于一个任意数的2次幂



对于一个上限不超过 `max` 的任意大小的数 `n`，可以通过O(1)的方法，求得一个值 `k` 使得 `k>=n` 且  `k%2==0`



方法： 

设max的 【二进制值】的位数为 `Q` ， n的 【二进制值】为 `M`



 则让 `n=n-1` 不停的 `n |= n>>>p` ，其中(p为2的幂次方，且`p>=1`，`p`< `Q`) 

可以得到从 `M`的第一个1起，右侧全为1的数 `R` 。

则所求结果  `K=R+1`





例如： `max` 为 `1<<30`  求 `n=65537` 的 `K`



由于max为 1<<30 所以， `Q=30` 则 p最大值为16 。     //(若max为 1<<65 ,则`Q=65`, p最大值为 64)

故：

```java

public int get(){
    n=n-1;
    n |= n>>>1;
    n |= n>>>2;
    n |= n>>>4;
    n |= n>>>8;
    n |= n>>>16;
    return n+1;
}
```
