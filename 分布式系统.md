分布式系统





# 1.什么是分布式系统



核心是  通过网路使一群计算机相互通信来完成一些连贯任务。



## 1.1 为什么使用分布式系统



使用多台计算机协作，以此获得



高性能   high Performance



并行   parallelism    

```
多台计算机 , 多个磁盘 并行运行。
```



容灾   fault  tolerance

```
多台计算机完成相同的任务。 一台宕机,其他的仍然工作
```



物理上的分布

```
例如 不同地区的机房
```





## 1.2 分布式系统遇到的问题



可能会遇到并发编程以及各种复杂交互带来的问题。

### 1.2.1 对时间的控制

例如:  对相关东西的时间控制， 超时机制，熔断机制



### 1.2.2  局部故障模式

局部故障通常让人难以提前预料。

```
如果是单机电脑，如果他宕机或者是断电。那么它将完全不可用。
但是如果是分布式系统，它可能是部分有用，部分故障，而其他继续工作。
```



### 1.2.3 性能并不容易提高

通常分布式系统的提出，都是为了获得 高性能 （high performance）。

使用一千台计算机来获得 n千倍的性能提升，通常是难以实现的。需要使用精细且巧妙的设计，才有可能达到。
