# 8.异常控制流

## 8.1. 用户态 内核态



### 1.1.1 为什么要有用户态内核态(作用)



现代OS的各种应用程序依赖于 `进程` 这个概念。`进程`为程序提供了隔离的各种运行条件（包括内存，设备等）

为了提供`进程`的隔离条件，需要依赖于 `用户态/内核态`。

OS必须通过`用户态`来限制应用可以访问的`指令` ,`内存地址`。





运行在`用户模式`的进程，不允许执行`特权指令`（例如 停止处理器，发起一个I/O操作，读文件等）

运行在`内核模式`的进程允许执行`指令集`中的全部指令，访问任意地址的内存。











## 8.2  异常控制流



我们通常把处理器将要执行的  `a0 ,a1 ,a2...ak` k条指令抽象为一个 `指令流`。指令流由`程序计数器`（程序计数器指向下一条将要执行指令的地址）来决定。

指令由 `ak 到 ak+1` 称为控制转移，这样的`控制转移序列` 称为 `控制流`。



`平滑`的控制流通常是连续的，但也可以通过 `jump`指令跳转到其他内存地址，来实现`控制转移`。





一个合格的OS必须应对 `当前系统状态`而做出`行为`改变（当一次网络IO事件完成，网络适配器发生一次IO事件中断，此时处理器需要响应本次中断）。



### 8.2.1 异常中断

异常。既可以由硬件实现，也可以由软件实现。

例如，`完成一次IO事件` `缺页中断` `硬件定时器发出一个信号` 都称为异常。





### 8.2.2 异常处理程序

异常处理程序(Exception handler)。 事实上它就是一段儿 专门处理`某种异常`的指令流。





例如 `缺页中断`异常，处理它需要从磁盘中加载缺页页块到内存中，当处理器发生`缺页中断`异常时，转而执行对应的异常处理程序（从磁盘中加载页块）加载完毕以后，恢复异常中断。

### 8.2.3 异常的类别



异常分为4种：`中断` `陷阱` `故障` `终止`。

陷阱： 例如在用户态希望执行特权指令，`系统调用`









## 8.3 进程

`进程`是 计算机科学中最深刻，最成功的概念之一。

什么是进程？ `进程` 是一个执行中程序的实例。





现代OS提供给`进程`的运行时特点： 好像OS只运行一个当前进程，好像当前进程独立的占有处理器和设备。（隔离）





### 8.3.1 进程上下文

什么是进程上下文？

```
进程上下文包含了程序正确运行所需的条件，包括代码和数据。 例如寄存器中的内容，程序计数器，使用的文件描述符
```





### 8.3.2 进程给程序的抽象



进程给应用程序提供了如下关键抽象：

```
1.独立的逻辑控制流。 好像程序独立占有处理器
2.独立的内存地址空间， 程序独占使用内存系统。
```
